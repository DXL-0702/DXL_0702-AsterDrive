# 邮件配置

只要你准备使用下面这些能力，就要先把邮件配置好：

- 公开注册后的邮箱激活
- 登录页里的找回密码
- 用户在 `设置 -> 安全` 里改绑邮箱
- 管理员自己给当前配置发测试邮件

后台入口：

```text
管理 -> 系统设置 -> 邮件投递
```

## 先做哪几步

推荐顺序很简单：

1. 填 SMTP 服务器、端口和加密方式
2. 按需填写用户名和密码
3. 填发件人地址和发件人名称
4. 发送一封测试邮件给自己
5. 再去测试注册激活、密码重置和邮箱改绑

不要先开放公开注册，再回头补邮件。  
不然用户账号已经创建出来了，却收不到激活邮件，只会卡在“等待激活”。

## 这些字段各自负责什么

| 字段 | 用途 |
| --- | --- |
| `mail_smtp_host` | SMTP 服务器地址 |
| `mail_smtp_port` | SMTP 端口，默认 `587` |
| `mail_security` | 是否启用加密；`465` 常见于隐式 SSL/TLS，其他端口通常走 STARTTLS |
| `mail_smtp_username` | SMTP 登录用户名 |
| `mail_smtp_password` | SMTP 登录密码 |
| `mail_from_address` | 收件人看到的发件邮箱 |
| `mail_from_name` | 收件人看到的发件人名称 |

如果你的 SMTP 服务不需要认证，可以把用户名和密码都留空。  
如果需要认证，这两个字段要一起填，不要只填一个。

## 怎么确认邮件真的能发出去

`邮件投递` 页面里有 `发送测试邮件` 按钮。

常见用法：

- 直接发给当前登录管理员的邮箱
- 临时改成另一个收件地址，确认外部邮箱也能收到

测试成功后，再继续做下面两件事：

1. 到登录页试一次“注册并收激活邮件”或“找回密码”
2. 到 `管理 -> 系统设置 -> 站点配置` 确认 `公开站点地址` 已经填对

## 邮件模版现在能改什么

当前内置了 5 组邮件模版：

- 注册激活
- 邮箱改绑确认
- 密码重置
- 密码重置结果通知
- 旧邮箱变更通知

每组模版都可以分别修改：

- 主题
- HTML 正文

页面里会提供对应模版可用的魔法变量。  
直接照着右侧提示填就行，不需要自己猜变量名。

## `公开站点地址` 为什么也要一起配

邮件里的激活链接、密码重置链接和邮箱改绑确认链接，都要生成能从外部打开的地址。

所以如果你的实际访问地址是：

```text
https://drive.example.com
```

就要到这里一起设置：

```text
管理 -> 系统设置 -> 站点配置 -> 公开站点地址
```

不要填带路径的地址，只填站点根地址。

## 没配好时会出现什么

最常见的结果有这些：

- 新用户能提交注册，但一直收不到激活邮件
- 找回密码按钮能点，但邮箱里没有重置链接
- 用户能发起邮箱改绑，但新邮箱迟迟收不到确认邮件
- 测试邮件失败

这时先做这几件事：

1. 检查 SMTP 主机、端口、加密方式和账号密码
2. 检查发件邮箱地址是否被 SMTP 服务允许
3. 检查 `公开站点地址` 是否已经改成外部真实域名
4. 在收件箱和垃圾箱里都确认一次
5. 邮件恢复正常后，再重新发送激活邮件或改绑确认邮件
