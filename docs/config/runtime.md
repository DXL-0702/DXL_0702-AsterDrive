# 系统设置

系统设置是管理员在后台直接维护的全站规则。

入口：

```text
管理 -> 系统设置
```

大多数改动不需要重启服务，保存后就会影响后续新请求、后续新上传或后续新发出的邮件。

## 管理员最常改的地方

| 你想做什么 | 去哪里改 |
| --- | --- |
| 让分享链接、邮件链接、WebDAV 地址和在线预览都指向正确域名 | `站点配置 -> 公开站点地址` |
| 改登录页和分享页显示的标题、Logo、favicon | `站点配置` |
| 给 Office 文件增加外部预览或 WOPI 打开方式 | `站点配置 -> 预览应用` |
| 关闭公开注册 | `用户管理 -> 允许公开注册新用户` |
| 改新用户和新团队的默认配额 | `存储与保留 -> 新用户默认存储配额` |
| 调整 Cookie 安全要求、Access Token 和 Refresh Token 有效期 | `认证与 Cookie` |
| 调整激活邮件、邮箱改绑和密码重置链接有效期 | `认证与 Cookie` |
| 配 SMTP、发测试邮件、改事务邮件模版 | `邮件投递` |
| 调整回收站、历史版本、团队归档和任务产物保留时间 | `存储与保留` |
| 关闭 WebDAV | `WebDAV -> 启用 WebDAV` |
| 调整邮件派发、后台任务派发和周期清理频率 | `运行与调度` |
| 开启或关闭审计日志 | `审计日志` |

## 先认识这些分组

当前页面会按下面这些分组显示：

- `站点配置`：公开站点地址、标题、Logo、favicon，以及在线预览相关设置
- `用户管理`：公开注册、注册激活、头像和 Gravatar
- `认证与 Cookie`：Cookie 安全规则、Token 有效期、激活 / 改绑 / 重置链接有效期
- `邮件投递`：邮件服务器（SMTP）、发件人、测试邮件和邮件模版
- `网络访问`：浏览器跨站访问规则（CORS）
- `运行与调度`：邮件队列、后台任务、周期清理、底层文件一致性检查和列表上限
- `存储与保留`：回收站、历史版本、默认配额、任务产物和缩略图限制
- `WebDAV`：是否对外提供 WebDAV
- `审计日志`：审计日志开关和保留时间
- `自定义配置`、`其他`：高级场景才需要碰

## 各分组里最值得先看的项

### 站点配置

如果你的站点要对外访问，先把这里配对。

- `公开站点地址`
  这里要填用户真正访问站点时使用的 HTTPS 地址，例如 `https://drive.example.com`。分享页、密码重置邮件、WebDAV 地址、Office / WOPI 预览和后续回调都会用它生成绝对链接。
- `站点标题`、`站点描述`
  影响登录页、分享页和应用页的标题与描述。
- `favicon`、浅色 Logo、深色 Logo
  影响浏览器标签页、登录页和站点头部的品牌显示。
- `预览应用`
  如果你要给 Office、PDF、表格或其他文件提供额外的“打开方式”，就在这里维护公开预览应用。内置预览器、外部 URL 模板和 WOPI 打开方式都在这里统一管理。
- `WOPI 相关时效`
  只有在你接入 OnlyOffice 一类在线 Office 预览或编辑服务时才需要调整。普通部署保持默认即可。

如果你准备启用 WOPI，推荐按这个顺序做：

1. 先填对 `公开站点地址`
2. 再到 `预览应用` 里启用已有应用，或通过 `WOPI Discovery` 导入新应用
3. 如果外部 Office 服务和 AsterDrive 不在同一个来源，再去 `网络访问` 里放行对应域名
4. 用真实的 `docx`、`xlsx` 或 `pptx` 文件试开一次，确认能正常保存回 AsterDrive

`WOPI access token 有效期`、`WOPI 锁有效期` 和 `WOPI discovery 缓存时长` 也都在这一组里。  
只有你已经接了 WOPI 服务，而且确实遇到会话过期或 discovery 更新不及时这类问题时，才需要手动调整。

### 用户管理

这一组主要控制账号入口和头像相关行为。

- `允许公开注册新用户`
  关闭后，登录页只保留已有账号登录和管理员初始化；后续新增账号只能由管理员创建。
- `注册后要求邮件激活`
  开启后，公开注册的普通用户必须先点激活邮件才能登录。
- `头像目录`
  用户上传头像会写到这个本地目录。相对路径会解析到服务端 `./data` 下面。
- `头像上传大小上限`
  超过这里的头像文件会直接被拒绝。
- `Gravatar 基础地址`
  如果当前网络访问官方 Gravatar 不稳定，可以换成代理或镜像地址。

### 认证与 Cookie

这一组决定浏览器登录体验和会话安全。

- `认证 Cookie 仅 HTTPS 发送`
  正式环境应该保持开启。只有本地或内网纯 HTTP 试跑时，才临时关闭。
- `访问令牌有效期`、`刷新令牌有效期`
  控制登录态维持多久。
- `注册激活链接有效期`
- `邮箱改绑链接有效期`
- `密码重置链接有效期`
- `验证邮件重发冷却时间`
- `密码重置请求冷却时间`

如果你只是正常部署，大多数时候只需要确认 Cookie 安全要求和链接时效是不是符合你的站点策略。

### 邮件投递

这组设置决定注册激活、密码重置和邮箱改绑邮件能不能发出去。

最常改的是：

- SMTP 主机、端口、用户名、密码
- 发件人地址、发件人名称
- 是否启用 SMTP 加密
- 测试邮件
- 注册激活、密码重置、邮箱改绑相关邮件模版

如果站点要开放注册、找回密码或邮箱改绑，邮件配置和 `公开站点地址` 要一起检查。

### 网络访问

这一组主要是浏览器跨站访问规则（CORS）。

只有在下面这些场景里，你通常才需要改它：

- 浏览器页面和 AsterDrive 不在同一个域名下
- 你要让别的站点在浏览器里直接调用 AsterDrive
- 你接入的 WOPI / Office 服务和 AsterDrive 不在同一个来源

大多数“前端页面和接口都在同一个站点里”的部署，不需要碰这里。

如果你接的是外部 WOPI 服务，最常见的做法就是把那个 Office 域名加入允许跨域来源。  
否则预览入口能显示，真正打开时也可能因为浏览器跨域限制失败。

### 运行与调度

管理员平时在这里决定后台任务的节奏。

默认行为大致是：

- 邮件队列每 5 秒扫描一次
- 后台任务队列每 5 秒扫描一次
- 周期性清理每 1 小时跑一次
- 底层文件一致性检查每 6 小时跑一次

还可以在这里调整：

- 团队成员列表单页上限
- 任务列表单页上限

如果你没有明显的性能或队列积压问题，通常保持默认就够了。

### 存储与保留

这一组最常决定“数据留多久”和“新对象默认拿多少空间”。

默认规则包括：

- 单文件历史版本上限：`10`
- 回收站保留时长：`7` 天
- 团队归档保留时长：`7` 天
- 任务保留时长：`24` 小时
- 新用户默认存储配额：`0`，表示不限制
- 缩略图源文件大小上限：`64 MiB`

需要注意：

- `新用户默认存储配额` 当前会同时影响新创建的用户和新创建的团队
- 这个设置只影响之后新创建的对象，已经存在的用户和团队不会被自动回写

### WebDAV

这里目前最重要的就是：

- `启用 WebDAV`

关闭后，桌面客户端会立刻无法继续通过 WebDAV 访问文件。  
如果你只是想改 WebDAV 路径前缀或 WebDAV 上传体积硬上限，那不是在这里改，而是在 [`config.toml` 的 `[webdav]`](/config/webdav) 里改。

### 审计日志

这一组决定后台和关键操作是否留下记录。

- `启用审计日志`
- `审计日志保留时长`

如果你希望之后还能追查“谁删了文件、谁创建了分享、谁改了团队成员”，就不要轻易把它关掉。

## 修改后什么时候生效

- 站点地址、标题、Logo、favicon：刷新页面或重新打开页面后按新值显示
- 预览应用和在线 Office 预览 / 编辑相关设置：之后新打开的预览按新规则生效
- WOPI access token、WOPI 锁和 discovery 缓存相关设置：之后新开的 WOPI 会话按新规则使用
- 公开注册、注册激活和邮件模版：之后新进入登录流程、新发出的邮件按新规则生效
- Cookie 安全策略和 Token 有效期：之后新登录、刷新和分享密码验证按新规则生效
- 头像目录和头像上传大小上限：之后新上传头像按新规则生效
- 默认配额：只影响之后新创建的用户和团队
- 回收站、团队归档、任务产物和审计日志保留时间：后台清理任务会按新规则工作
- 历史版本上限：之后新产生版本时按新规则裁剪
- 缩略图源文件大小上限：之后新进入缩略图任务的文件按新规则判断
- 邮件派发、后台任务派发和周期清理频率：之后的后台轮询按新规则执行
- WebDAV 开关、跨站访问规则：新请求立即按新规则响应

## 关于“自定义配置”

如果你只是正常维护 AsterDrive，优先使用内置分组里的设置就够了。

`自定义配置` 更适合：

- 你已经很清楚自己要覆盖哪个内部配置键
- 当前版本里暂时还没有现成的可视化表单

不确定时，不要先从这里下手。
