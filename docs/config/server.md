# 服务器配置

```toml
[server]
host = "127.0.0.1"
port = 3000
workers = 0
```

这一组配置决定 AsterDrive 监听哪个地址、哪个端口，以及使用多少工作线程。
大多数部署只会改 `host`，有时会顺手改 `port`。

## 常见场景

- 本机测试：一般保持默认即可
- Docker 或容器：通常把 `host` 改成 `0.0.0.0`
- 反向代理部署：端口通常继续用 `3000`
- 不确定线程数时：先保持 `workers = 0`

## 字段说明

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `host` | `"127.0.0.1"` | 绑定地址；容器部署通常设为 `0.0.0.0` |
| `port` | `3000` | HTTP 监听端口 |
| `workers` | `0` | 工作线程数；`0` 表示自动选择 |

## 最常见的写法

### 本机测试

```toml
[server]
host = "127.0.0.1"
port = 3000
workers = 0
```

### Docker 或容器

```toml
[server]
host = "0.0.0.0"
port = 3000
workers = 0
```

## 使用建议

- 大多数用户不需要手动调整 `workers`
- 对外访问是否稳定，通常更依赖反向代理和 HTTPS 配置
- 如果你已经用了 Caddy 或 Nginx，应用本身继续监听内部端口即可

## 对应环境变量

```bash
ASTER__SERVER__HOST=0.0.0.0
ASTER__SERVER__PORT=3000
```
