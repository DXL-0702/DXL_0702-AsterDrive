# 管理后台

第一个创建成功的账号会自动成为管理员。

## 入口

管理员登录后，可以在右上角用户菜单进入 `管理`。后台里可以完成这些工作：

- 管理用户和配额
- 管理本地或 S3 存储策略
- 调整系统设置
- 查看和删除全站分享
- 处理卡住的资源锁
- 查看审计日志

## 用户管理

用户页面里可以直接处理：

- 创建用户
- 调整角色和状态
- 修改总配额
- 删除用户
- 给用户分配可用的存储策略

如果你不想让新同事自己从登录页注册，也可以直接由管理员在这里创建账号。

总配额控制的是这个用户一共能用多少空间。用户的上传和已用空间也会显示在这里。

分配存储策略时，你可以：

- 指定这个用户可以使用哪些策略
- 为用户选择默认策略
- 为某条分配单独设置额度

系统会保护首个管理员账号，避免你误把唯一管理员停用或删除。

## 存储策略

存储策略决定两件事：

- 文件实际存到哪里
- 上传时走哪种方式

当前可以管理两类策略：

- `local`：存到本地目录
- `s3`：存到 S3 或兼容对象存储

你可以在这里：

- 创建和编辑策略
- 设为系统默认策略
- 测试策略连接
- 控制单文件大小上限
- 控制分片大小
- 为 S3 策略开启 Presigned 直传

已经有文件在用的策略，不要直接修改 `base_path`、`bucket`、`endpoint` 这类决定存储位置的字段。要改位置，先迁移旧数据，再切换到新策略。

## 系统设置

系统设置用于调整全站运行行为，常见项目包括：

- 是否启用 WebDAV
- 回收站保留天数
- 单文件保留多少历史版本
- 新用户默认配额
- 是否记录审计日志
- 审计日志保留天数
- Gravatar 头像地址

大多数部署只需要维护这些内置项。  
页面里还能新增“自定义配置”，但那通常只给高级集成或定制场景使用，不是日常必需项。

如果你的服务器所在网络访问不到官方 Gravatar，可以把 `gravatar_base_url` 改成你可访问的镜像地址，这样用户选择 Gravatar 头像时才会正常显示。

## 分享管理

分享页面可以查看全站所有分享链接，并随时删除其中任意一条。

常见用法：

- 某个公开链接不该继续访问
- 某个分享已经过期但还想提前回收
- 统一检查当前有哪些公开链接仍在使用

## 锁管理

锁页面主要用于排障。常见场景：

- 某个文件一直显示被锁定
- 某个 WebDAV 客户端异常退出后没有释放锁
- 需要集中清理过期锁

管理员可以在这里查看锁、强制解锁，或者清理过期锁。

## 审计日志

审计日志页面用于查看关键操作记录。常见用途是：

- 查谁删除了某个文件
- 查某个用户什么时候登录或修改过内容
- 配合分享、锁、WebDAV 问题做排查

是否记录审计日志，以及日志保留多久，都由系统设置控制。

## 管理员日常检查清单

建议定期确认下面几项：

1. 用户总配额是否合理
2. 默认存储策略是否仍然正确
3. 回收站和历史版本保留时间是否符合当前容量
4. 是否存在不该继续公开的分享链接
5. 是否有长期未释放的锁
6. 审计日志是否开启，保留时间是否足够
7. 如果用户使用 Gravatar，头像地址是否在当前网络可访问
