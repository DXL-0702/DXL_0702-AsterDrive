# 管理后台

第一个创建成功的账号会自动成为管理员。

## 从哪里进入

管理员登录后，可以在右上角用户菜单进入 `管理`。
后台左侧菜单当前包括:

- 概览
- 用户
- 团队
- 存储策略
- 策略组
- 分享
- 锁
- 系统设置
- 审计日志
- 关于

## 概览

概览页适合先看全站当前状态。这里可以看到:

- 用户总数、启用中用户、已禁用用户
- 文件数量和总容量
- 分享数量
- 最近 7 天趋势
- 最近活动

如果你关闭了审计日志，趋势和最近活动会明显变少，甚至显示为空。

## 用户

`用户` 页面用于管理账号、角色、状态、总配额和策略组绑定。

你可以在这里:

- 创建用户
- 搜索和筛选用户
- 调整角色和状态
- 修改总配额
- 打开用户详情

在用户详情里，还可以继续做这些事:

- 重置这个用户的登录密码
- 强制让这个用户当前所有设备重新登录
- 给这个用户绑定一个可用策略组

系统会保护初始管理员账号，避免你误把唯一的管理员停用、降级或删除。

## 团队

`团队` 页面用于创建、归档、恢复和查看团队工作空间。

你可以在这里:

- 创建团队
- 指定初始团队管理员
- 给团队绑定一个可用策略组
- 查看成员数、空间占用和归档状态
- 打开团队详情页继续查看成员和团队审计

普通团队管理员自己的日常入口在 `设置 -> 团队`。
系统管理员的全局团队管理入口在这里。

## 存储策略

存储策略决定两件事:

- 文件最终存到哪里
- 上传时走哪种方式

当前后台支持两类策略:

- `local`: 存到本地目录
- `s3`: 存到 S3 或兼容对象存储

在这里你可以:

- 创建和编辑策略
- 测试连接
- 设为系统默认策略
- 控制单文件大小上限
- 控制分片大小
- 为 S3 选择上传方式

已经有文件在用的策略，不要直接修改 `base_path`、`bucket`、`endpoint` 这类决定实际存放位置的字段。要改位置，先迁移旧数据，再切换到新策略。

## 策略组

策略组决定“用户或团队上传时，应该走哪条存储策略”。

最常见的做法是:

- 默认策略组只有一条规则，全部文件都走默认本地策略
- 同时使用本地和 S3 时，按文件大小分成多条规则
- 把不同用户或团队绑定到不同策略组

如果你需要改上传路线，通常顺序是:

1. 先准备好存储策略
2. 再配置策略组规则
3. 最后把用户或团队绑定到对应策略组

## 分享

`分享` 页面可以统一查看当前全站所有分享链接，并随时删除其中任意一条。

常见用法:

- 某个公开链接不该继续访问
- 某个分享已经不再需要
- 想检查目前还有哪些资料在公开访问

## 锁

`锁` 页面主要用于处理卡住的 WebDAV 锁或编辑锁。

常见场景:

- 某个文件一直显示被锁定
- 某个 WebDAV 客户端异常退出后没有释放锁
- 需要集中清理已过期的锁

管理员可以在这里查看锁、强制解锁，或者一次性清理过期锁。

## 系统设置

`系统设置` 用来维护全站运行规则。
页面里会按 WebDAV、存储、审计、网络和通用项目分组显示。

最常改的项目包括:

- WebDAV 开关
- 回收站保留天数
- 单文件保留多少历史版本
- 团队归档保留天数
- 新用户默认配额
- 是否记录审计日志
- 审计日志保留天数
- Gravatar 头像地址
- 跨域来源设置（只有你需要跨域访问时才改）

页面里也可以新增自定义配置，但大多数部署平时只需要维护内置设置。

## 审计日志

`审计日志` 页面用于查看关键操作记录。

常见用途:

- 查谁删除了某个文件
- 查某个用户什么时候登录或修改过内容
- 排查分享、锁和 WebDAV 相关问题

是否记录审计日志，以及日志保留多久，都由系统设置控制。

## 关于

`关于` 页面会显示当前部署版本、许可证、仓库和文档入口。
排查“现在到底跑的是哪个版本”时，先看这里。

## 管理员日常检查清单

建议定期确认下面几项:

1. 用户总配额是否合理
2. 默认存储策略和默认策略组是否仍然正确
3. 团队绑定的策略组是否仍可用
4. 回收站、历史版本和团队归档保留时间是否符合当前容量
5. 是否存在不该继续公开的分享链接
6. 是否有长期未释放的锁
7. 审计日志是否开启，保留时间是否足够
8. Gravatar 头像地址在当前网络里是否可访问
