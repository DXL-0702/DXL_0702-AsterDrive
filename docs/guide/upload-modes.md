# 上传与大文件

AsterDrive 不要求用户手动切换上传方式。
你只需要上传文件，系统会根据文件大小和当前存储策略自动选择合适的方式。

## 日常使用时需要知道的事

- 小文件通常直接完成上传
- 大文件通常会自动分片上传
- 如果管理员把存储策略设成 S3 / MinIO，并开启了直传，浏览器会直接把文件传到对象存储
- 上传中断后，浏览器可能提示继续未完成的上传；重新选择同一个文件即可继续

对普通用户来说，知道这些就够了。你不需要手动选择“普通上传”还是“分片上传”。

## 管理员最常调的三个地方

### 存储策略

上传行为由当前命中的存储策略决定。管理员最常改的是：

- 存储驱动：本地或 S3
- 单文件大小上限
- 分片大小
- 是否启用 Presigned 直传

### 反向代理

- 如果上传流量经过 AsterDrive，就要检查代理层的请求体大小和超时
- 如果用户经常上传大文件，代理层不要把上传限制设得太小

### S3 / MinIO 的 CORS

如果你启用了 Presigned 直传，对象存储必须允许浏览器直接上传，否则用户会在上传阶段直接失败。

## 如果日志或报错里出现上传模式名称

| 名称 | 说明 |
| --- | --- |
| `direct` | 小文件直接通过 AsterDrive 上传 |
| `chunked` | 大文件分片通过 AsterDrive 上传 |
| `presigned` | 小文件直接上传到 S3 / MinIO |
| `presigned_multipart` | 大文件分片直接上传到 S3 / MinIO |

这些名字主要用于排错。普通用户平时不需要关心它们。

## 上传失败时先查这几项

1. 当前命中的存储策略是什么
2. 该策略的分片大小和 Presigned 直传是否符合预期
3. 代理层是否限制了上传体积或超时时间
4. 如果是 `presigned` 或 `presigned_multipart`，对象存储是否已经正确配置 CORS
5. 用户配额和单文件大小上限是否触发限制
