# 快速开始

按下面的顺序做，你可以在一台新机器上把 AsterDrive 跑起来，并完成第一轮验收。

## 1. 先把服务启动起来

第一次试用，最省事的方式是直接运行官方镜像：

```bash
docker run -d \
  --name asterdrive \
  -p 3000:3000 \
  -e ASTER__SERVER__HOST=0.0.0.0 \
  -e ASTER__AUTH__BOOTSTRAP_INSECURE_COOKIES=true \
  -e ASTER__DATABASE__URL="sqlite:///data/asterdrive.db?mode=rwc" \
  -v asterdrive-data:/data \
  ghcr.io/apts-1547/asterdrive:latest
```

这条命令适合本机、内网或临时纯 HTTP 试跑环境。

- 如果你已经准备好通过 HTTPS 对外访问，可以去掉 `ASTER__AUTH__BOOTSTRAP_INSECURE_COOKIES=true`
- 正式上线时，建议保持 `bootstrap_insecure_cookies = false`，并让后台系统设置里的 Cookie 安全开关保持开启

首次成功启动后，AsterDrive 会自动完成下面这些准备：

- 在当前工作目录的 `data/` 下生成 `config.toml`
- 创建或连接数据库，并自动更新数据库结构
- 创建默认本地存储策略 `Local Default`
- 创建默认策略组 `Default Policy Group`
- 创建本地上传目录 `data/uploads`
- 创建临时目录 `data/.tmp` 和 `data/.uploads`
- 初始化后台默认设置，并启动必要的后台任务

如果你使用官方 Docker 镜像并保持默认相对路径：

- 数据库和上传目录会落到容器里的 `/data`
- `config.toml` 会生成在容器里的 `/data/config.toml`

打开地址：

```text
http://服务器地址:3000
```

## 2. 创建第一个管理员账号

浏览器打开站点后，登录页会根据你输入的用户名或邮箱自动判断流程：

- 系统里还没有任何用户：创建管理员账号
- 输入的是已有账号：登录
- 输入的是新账号，而且管理员允许公开注册：注册普通账号

第一个创建成功的账号会自动成为管理员。

第一个管理员账号是直接创建并立即可用的。  
后续普通用户如果通过公开注册创建账号，需要先完成邮箱激活。

如果你准备直接对公网开放服务，至少先确认下面两件事：

1. 公开注册是否真的要开启
2. 邮件投递和公开站点地址是否已经配置好

## 3. 先做一轮最基础的可用性检查

管理员登录后，先在“我的云盘”里完成这几步：

1. 新建一个测试文件夹
2. 上传一个小文件
3. 打开文件，确认可以预览、编辑或下载
4. 把文件删除到回收站
5. 再从回收站恢复回来

这几步都能正常完成，说明浏览器端、数据库和默认存储路线已经基本可用。

## 4. 试一次分享

在文件或文件夹的操作菜单里创建分享链接，并按需要设置：

- 密码
- 过期时间
- 最大下载次数

把链接发到无痕窗口、手机或另一台设备里，确认公开页面能正常打开。

## 5. 如果要用 WebDAV，再做一次真实连接

如果你准备让 Finder、Windows 资源管理器、rclone 或同步工具接入：

1. 打开左侧 `WebDAV`
2. 创建一个专用 WebDAV 账号
3. 复制 WebDAV 地址、用户名和密码
4. 在客户端里做一次读写测试

密码只会在创建成功时显示一次，建议立刻保存到密码管理器。

## 6. 管理员第一次登录后至少检查这些地方

- `管理 -> 概览`
- `管理 -> 用户`
- `管理 -> 团队`
- `管理 -> 存储策略`
- `管理 -> 策略组`
- `管理 -> 系统设置`
- 左侧 `WebDAV`

重点确认：

- 默认存储策略和默认策略组已经创建
- 新用户默认配额是否合适
- 新创建团队要用的默认配额和默认策略组是否合适
- 如果站点准备对外访问，`公开站点地址` 是否已经填成真实域名
- 如果要开放注册、找回密码或邮箱改绑，测试邮件是否已经发通
- 如果要接 OnlyOffice 一类 WOPI 服务，`站点配置 -> 预览应用` 和 `网络访问` 是否已经一起配好
- 回收站保留天数、历史版本数量和团队归档保留天数是否符合预期
- WebDAV 是否需要保持开启
- 文件是否继续落本地，还是要改到 S3 / MinIO
- Gravatar 头像地址在当前网络里是否可访问

## 7. 正式上线前做一轮验收

- 浏览器可以正常登录和退出
- 文件可以上传、下载、删除到回收站并恢复
- 分享链接可以正常打开
- `管理 -> 存储策略` 和 `管理 -> 策略组` 都能正常打开
- 如果启用了外部 Office / WOPI 打开方式，至少用一个真实 Office 文件试开并保存一次
- 如果你要用团队空间，至少创建一个团队并验证成员能进入对应工作空间
- `http://服务器地址:3000/health` 返回正常
- `http://服务器地址:3000/health/ready` 返回正常
- 如果启用了 WebDAV，客户端可以成功连接并读写

通过这轮检查后，再继续做域名、HTTPS、反向代理、备份和升级安排。
