# 快速开始

这份快速开始只做一件事:
先把 AsterDrive 跑起来，再确认管理员、上传、分享和 WebDAV 都能正常工作。

## 1. 先把服务启动起来

第一次试用，最省事的方式是直接运行官方镜像:

```bash
docker run -d \
  --name asterdrive \
  -p 3000:3000 \
  -e ASTER__SERVER__HOST=0.0.0.0 \
  -e ASTER__AUTH__COOKIE_SECURE=false \
  -e ASTER__DATABASE__URL="sqlite:///data/asterdrive.db?mode=rwc" \
  -v asterdrive-data:/data \
  ghcr.io/apts-1547/asterdrive:latest
```

这条命令适合本地或内网里的纯 HTTP 试跑环境。
如果你已经准备好用 HTTPS 对外提供服务，把 `ASTER__AUTH__COOKIE_SECURE=false` 去掉，或者在配置文件里保持 `cookie_secure = true`。

第一次成功启动后，AsterDrive 会自动完成这些准备工作:

- 在当前工作目录生成 `config.toml`
- 创建或连接数据库
- 自动更新数据库结构
- 创建默认本地存储策略 `Local Default`
- 创建默认策略组 `Default Policy Group`
- 创建默认上传目录 `data/uploads`
- 创建临时目录 `data/.tmp` 和 `data/.uploads`
- 初始化系统设置

使用官方 Docker 镜像并保持默认相对路径时，这些默认目录都会落在容器里的 `/data` 下面。

默认访问地址:

```text
http://服务器地址:3000
```

## 2. 打开页面并创建第一个账号

1. 在浏览器打开 `http://服务器地址:3000`
2. 在登录页输入用户名或邮箱
3. 页面会自动判断当前应该执行哪一种流程

常见情况:

- 系统里还没有用户时，创建管理员
- 输入的是已有账号时，直接登录
- 输入的是新账号时，创建普通账号

第一个创建成功的账号会自动成为管理员。

## 3. 先做一轮最基础的文件检查

登录后先做这几步:

1. 新建一个测试文件夹
2. 上传一个小文件
3. 打开它，确认可以预览、编辑或下载
4. 把它移到回收站
5. 再从回收站恢复回来

只要这几步正常，说明浏览器端、数据库和默认存储路线已经基本可用。

## 4. 试一次分享

在文件或文件夹的操作菜单里创建分享链接，并按需要设置:

- 密码
- 过期时间
- 最大下载次数

把链接发到无痕窗口、手机或另一台设备里，确认公开页面能正常打开。

## 5. 如果要用 WebDAV，再做一次连接测试

如果你要让 Finder、Windows 资源管理器、rclone 或同步工具接入:

1. 打开左侧 `WebDAV`
2. 创建一个专用 WebDAV 账号
3. 复制 WebDAV 地址、用户名和密码
4. 在客户端里做一次真实登录和读写测试

密码只会在创建成功时显示一次，建议立刻保存到密码管理器。

## 6. 第一次登录后，管理员至少检查这几处

- `管理 -> 概览`
- `管理 -> 用户`
- `管理 -> 团队`
- `管理 -> 存储策略`
- `管理 -> 策略组`
- `管理 -> 系统设置`
- 左侧 `WebDAV`

重点确认:

- 默认存储策略是否正确
- 默认策略组是否正确
- 新用户默认配额是否合适
- 回收站保留天数、历史版本数量和团队归档保留天数是否合适
- WebDAV 是否需要保持开启
- 文件是继续放本地磁盘，还是改到 S3 / MinIO
- 如果你要用 Gravatar，头像地址在当前网络里是否能访问

## 7. 正式上线前，再做一次验收

- 浏览器可以正常登录和退出
- 文件可以上传、下载、移动到回收站并恢复
- 分享链接可以正常打开
- 如果你要用团队空间，至少创建一个团队并验证成员能进入对应工作空间
- `http://127.0.0.1:3000/health` 返回正常
- `http://127.0.0.1:3000/health/ready` 返回正常
- 如果启用了 WebDAV，客户端可以成功连接并读写

通过这轮检查后，再继续做域名、HTTPS、反向代理和备份安排。
