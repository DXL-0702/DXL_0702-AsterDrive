# 文件编辑

## 哪些文件适合直接在网页里编辑

浏览器内编辑主要面向文本类文件，例如:

- Markdown
- CSV / TSV
- JSON
- XML
- 配置文件
- 日志
- 脚本
- 常见代码文件

二进制文件、Office 文档和需要多人实时协作的内容，不适合用这个编辑器处理。

## 网页编辑时会发生什么

打开并保存一个文本文件时，AsterDrive 会自动处理这些事情:

- 读取当前内容
- 在编辑期间为文件加锁
- 保存前检查文件是否已经被别人改过
- 保存成功后生成一个历史版本
- 关闭编辑器时自动释放锁

如果你保存时收到冲突提示，通常说明这份文件在你编辑期间已经被别人改过。先刷新内容，再决定是否继续。

## 版本历史怎么用

只要发生覆盖写入，就会生成历史版本。常见来源包括:

- 浏览器内文本编辑
- WebDAV 覆盖保存
- 其他会直接覆盖原文件内容的写入方式

你可以在版本历史里:

- 查看旧版本
- 恢复某个版本
- 删除某个版本

恢复旧版本后，该版本之后的历史版本会一起被截断，所以恢复前先确认。

## 通过 WebDAV 编辑也会保留版本

如果你更习惯桌面应用，也可以通过 WebDAV 编辑文件。
通过 WebDAV 覆盖保存时，同样会生成历史版本。

常见场景包括:

- Finder 挂载网络位置
- Windows 映射网络驱动器
- rclone 或其他同步工具
- 支持 WebDAV 的编辑器或办公软件

## 遇到锁定问题怎么办

文件正在被编辑时，其他用户不能随意覆盖、重命名、移动或删除它。

常见处理方法:

- 先让持有锁的用户正常保存并退出
- 如果是异常中断留下的残留锁，让管理员到 `管理 -> 锁` 里强制解锁

## 使用边界

- 适合文本类文件
- 不支持多人实时协作
- 不会自动合并冲突
- 历史版本保留数量由管理员控制
