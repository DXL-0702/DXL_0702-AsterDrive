# 文件编辑

## 可以直接编辑哪些文件

浏览器内编辑主要面向文本类文件，例如：

- Markdown
- CSV / TSV
- JSON
- XML
- 配置文件
- 日志
- 脚本和常见代码文件

## 编辑时会发生什么

打开并保存一个文件时，AsterDrive 会自动处理下面这些事：

- 打开文件时会读取当前内容
- 进入编辑时会加锁
- 保存时会检查文件是否已经被别人改过
- 保存成功后会生成一个历史版本
- 关闭编辑器时会自动解锁

如果你保存时收到冲突提示，通常说明文件已经被其他人改过，先刷新再决定是否继续编辑。

## 版本历史

只要发生覆盖写入，就会生成历史版本。常见来源包括：

- 浏览器内文本编辑
- WebDAV 覆盖保存
- 其他会覆盖原文件内容的写入方式

你可以在版本历史里：

- 查看旧版本
- 恢复某个版本
- 删除某个版本

恢复旧版本后，该版本及其之后的历史版本会被删除。

历史版本保留数量由管理员在系统设置里控制。

## WebDAV 编辑

如果你更习惯桌面应用，也可以通过 WebDAV 编辑文件。

典型场景包括：

- Finder 挂载网络位置
- Windows 映射网络驱动器
- rclone 或其他同步工具
- 某些支持 WebDAV 的编辑器或办公软件

通过 WebDAV 覆盖保存文件时，同样会生成历史版本。

## 锁定和解锁

文件正在被编辑时，可能会处于锁定状态。锁定期间，其他用户不能随意覆盖、重命名、移动或删除它。

遇到锁定问题时，通常这样处理：

- 先让持有锁的用户正常保存并退出
- 如果是异常中断导致的残留锁，管理员可以在后台强制解锁

## 当前边界

- 只适合文本类文件
- 不支持多人实时协作
- 不会自动合并冲突
- 恢复旧版本会截断该版本及其之后的历史版本，所以恢复前先确认
