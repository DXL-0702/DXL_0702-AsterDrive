# 常用流程

下面这些流程基本覆盖了 AsterDrive 最常见的使用场景。第一次部署后，照着做一遍，通常就能把系统跑顺。

## 1. 管理员完成初始检查

第一次登录后，先检查这四个地方：

1. `管理员后台 -> 用户`
2. `管理员后台 -> 存储策略`
3. `管理员后台 -> 系统设置`
4. `WebDAV`

建议至少确认：

- 默认存储策略是否正确
- 新用户默认配额是否合适
- 回收站保留天数和历史版本数量是否符合预期
- 是否需要启用 WebDAV，并为设备创建专用账号

## 2. 上传并整理文件

比较顺手的顺序通常是：

1. 创建几个基础文件夹
2. 上传文件
3. 用拖拽把文件移到目标目录
4. 用多选完成批量移动、复制或删除

如果上传的是大文件，AsterDrive 会自动切换到更适合的上传方式，不需要手动处理。

## 3. 分享文件或文件夹

常见做法是：

1. 在文件或文件夹上创建分享链接
2. 需要保密时设置密码
3. 需要时效控制时设置过期时间
4. 需要限流时设置下载次数

文件分享适合发送单个文件。文件夹分享适合让对方进入共享目录继续浏览和下载里面的文件。

## 4. 用 WebDAV 连接桌面客户端

如果你希望在 Finder、Windows 资源管理器、rclone 或同步工具里直接使用 AsterDrive，推荐这样做：

1. 在 `WebDAV` 页面创建一个专用账号
2. 按需要限制到某个目录
3. 把 WebDAV 地址、用户名和密码填进客户端
4. 先做一次真实读写测试

这样某台设备停用时，只需要停用这一个 WebDAV 账号，不影响网页登录。

## 5. 处理删除和恢复

普通删除后的项目会先进回收站。日常处理方式通常是：

1. 误删后直接去回收站恢复
2. 不再需要的项目在回收站里永久删除
3. 管理员定期确认回收站保留天数是否合理

如果某个项目已经没有恢复必要，也可以直接在回收站里彻底删除。

## 6. 管理员给新用户分配空间

比较常见的做法是：

1. 在 `用户` 页面设置用户总配额
2. 在 `存储策略` 页面创建本地或 S3 策略
3. 给用户分配可用策略，并指定默认策略

这样可以做到：

- 大部分用户走默认本地存储
- 某些用户单独走 S3
- 某些目录或账号有更大的上传空间
