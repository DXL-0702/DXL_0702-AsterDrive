# 常用流程

下面这些流程覆盖了 AsterDrive 最常见的日常动作。  
如果你不想先通读全部手册，直接按场景照着做就行。

## 新部署后的第一次管理员检查

第一次登录后，建议按这个顺序检查：

1. `管理 -> 概览`
2. `管理 -> 用户`
3. `管理 -> 团队`
4. `管理 -> 存储策略`
5. `管理 -> 策略组`
6. `管理 -> 系统设置`
7. 左侧 `WebDAV`

重点确认：

- 默认存储策略是否正确
- 默认策略组是否正确
- 新用户和新团队默认配额是否合理
- 如果站点准备对外访问，`公开站点地址` 是否已经填对
- 如果要开放注册、找回密码或邮箱改绑，测试邮件是否已经发通
- 如果要接 OnlyOffice 一类 WOPI 服务，`站点配置 -> 预览应用` 和 `网络访问` 是否已经一起配好
- 公开注册开关是否符合你的部署方式
- 回收站保留天数、历史版本数量和团队归档保留天数是否符合预期
- WebDAV 是否需要开启
- 如果要用 Gravatar，头像地址是否可访问

## 给 Office 文件接在线打开方式

如果你准备把 `docx`、`xlsx`、`pptx` 这类文件交给外部服务处理，推荐这样做：

1. 先把 `管理 -> 系统设置 -> 站点配置 -> 公开站点地址` 填对
2. 到 `管理 -> 系统设置 -> 站点配置 -> 预览应用` 里启用或导入对应 WOPI 应用
3. 如果外部 Office 服务和 AsterDrive 不在同一个来源，再到 `管理 -> 系统设置 -> 网络访问` 放行对应域名
4. 用一个真实 Office 文件试开并保存一次

## 上传并整理文件

整理资料时，通常按这个顺序做：

1. 先建立基础文件夹
2. 上传文件或文件夹
3. 用拖拽把它们移动到目标目录
4. 用多选批量完成移动、复制或删除

## 给不同用户或团队安排存储路线

如果你想让不同对象走不同的存储位置，推荐这样做：

1. 在 `管理 -> 存储策略` 里先创建真实的存储目标
2. 在 `管理 -> 策略组` 里定义文件大小分流规则
3. 在 `管理 -> 用户` 或 `管理 -> 团队` 里绑定对应策略组

常见做法：

- 所有人默认走本地存储
- 某些团队单独走 S3 / MinIO
- 小文件走本地，大文件走对象存储

## 创建团队空间

当前版本里，团队由系统管理员创建。  
推荐顺序：

1. 在 `管理 -> 团队` 里创建团队
2. 指定初始团队管理员
3. 选择这个团队要绑定的策略组
4. 让成员从左侧工作空间列表进入对应团队空间

团队创建完成后，团队里的管理员或所有者可以在 `设置 -> 团队` 里继续管理成员和查看团队审计。

## 发送资料给别人

发资料时，通常这样做：

1. 先切到正确的工作空间
2. 在文件或文件夹上创建分享链接
3. 需要保密时设置密码
4. 需要时效控制时设置过期时间
5. 需要限制传播时设置下载次数
6. 去 `我的分享` 里复制链接，并检查公开页是否正常

如果只是改密码、过期时间或下载次数，直接在 `我的分享` 里编辑原有链接即可。

## 继续未完成的上传

大文件上传中断后，常见处理方式是：

1. 回到原来的文件夹
2. 找到未完成的上传任务
3. 重新选择同一个文件

只要上传会话还没过期，系统会尽量继续未完成的部分，而不是从头开始。

## 给设备创建 WebDAV 账号

如果你要在 Finder、Windows 资源管理器、rclone 或同步工具里接入 AsterDrive，推荐这样做：

1. 在 `WebDAV` 页面创建一个专用账号
2. 按需要限制到某个目录
3. 把 WebDAV 地址、用户名和密码填进客户端
4. 先做一次真实读写测试

这样某台设备停用时，只需要停用这一条 WebDAV 账号，不影响网页登录密码。

## 处理误删

普通删除后的项目会先进入回收站。  
最常见的处理方式是：

1. 误删后直接去回收站恢复
2. 不再需要的项目在回收站里永久删除
3. 管理员定期确认回收站保留天数是否合理

## 处理锁定问题

如果文件一直显示被锁定，常见处理顺序是：

1. 先让正在编辑它的人正常保存并退出
2. 如果是 WebDAV 客户端异常退出，再去 `管理 -> 锁` 查看
3. 确认锁已经失效后，由管理员手动清理
