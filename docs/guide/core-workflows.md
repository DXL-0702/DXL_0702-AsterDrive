# 常用流程

## 新部署后的首次检查

第一次登录后，建议先检查下面四处：

- `管理 -> 用户`
- `管理 -> 存储策略`
- `管理 -> 系统设置`
- `WebDAV`

重点确认：

- 默认存储策略是否正确
- 新用户默认配额是否合适
- 回收站保留天数和历史版本数量是否符合预期
- 是否需要启用 WebDAV，并为设备创建专用账号
- 如果要用 Gravatar，头像地址是否可访问

## 上传并整理文件

日常整理资料时，通常这样做：

1. 创建几个基础文件夹
2. 上传文件
3. 用拖拽把文件移到目标目录
4. 用多选完成批量移动、复制或删除

## 发分享链接

发资料给别人时，通常这样做：

1. 在文件或文件夹上创建分享链接
2. 需要保密时设置密码
3. 需要时效控制时设置过期时间
4. 需要限流时设置下载次数

文件分享适合发送单个文件。文件夹分享适合让对方进入共享目录继续浏览和下载里面的文件。

## 给设备创建 WebDAV 账号

如果你要在 Finder、Windows 资源管理器、rclone 或同步工具里直接接入 AsterDrive，推荐这样做：

1. 在 `WebDAV` 页面创建一个专用账号
2. 按需要限制到某个目录
3. 把 WebDAV 地址、用户名和密码填进客户端
4. 先做一次真实读写测试

这样某台设备停用时，只需要停用这一个 WebDAV 账号，不影响网页登录。

## 处理误删和彻底删除

普通删除后的项目会先进回收站。常见处理方式是：

1. 误删后直接去回收站恢复
2. 不再需要的项目在回收站里永久删除
3. 管理员定期确认回收站保留天数是否合理

## 给新用户分配空间和策略

新增用户后，通常这样处理：

1. 在 `用户` 页面创建账号或找到现有账号
2. 设置用户总配额
3. 在 `存储策略` 页面创建本地或 S3 策略
4. 给用户分配可用策略，并指定默认策略

这样可以做到：

- 大部分用户走默认本地存储
- 某些用户单独走 S3
- 某些账号有更大的上传空间
