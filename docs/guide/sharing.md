# 分享与公开访问

AsterDrive 当前支持文件分享和文件夹分享，两者都可以生成公开访问链接。

前端公开页面路径：

```text
/s/:token
```

公开 API 路径：

```text
/api/v1/s/{token}
```

## 创建分享时可设置什么

创建分享时当前支持这些选项：

- 分享文件或文件夹
- 可选密码
- 过期时间
- 下载次数限制

同一个资源同一时间只允许存在一个活跃分享。如果需要新链接，要先删除旧分享，或者等待旧分享过期。

## 文件分享

文件分享当前提供：

- 公开访问页
- 文件下载
- 文件预览
- 可选密码校验
- 可选过期时间
- 可选下载次数限制

如果分享设置了密码，访问者需要先调用 `/verify`，服务端会写入一个短期有效的 HttpOnly Cookie。

## 文件夹分享

文件夹分享当前提供：

- 公开访问页
- 分享根目录的内容列表
- 根目录中展示出来的文件直接下载
- 根目录中展示出来的文件预览
- 可选密码校验
- 可选过期时间

## 当前能力边界

这块最容易被误解，直接说死：

- **已支持**：公开分享页展示分享根目录内容
- **已支持**：下载分享根目录中展示出来的子文件
- **未支持**：在公开分享页继续进入子文件夹浏览

也就是说，当前“子文件可下载”不等于“子文件夹可导航”。

## 密码保护

有密码的分享访问流程是：

1. 访问 `/api/v1/s/{token}` 获取公开信息
2. 调用 `/api/v1/s/{token}/verify` 提交密码
3. 服务端写入 `aster_share_{token}` Cookie
4. 再访问下载、缩略图或文件夹内容接口

如果分享本身没有密码，`/verify` 不会成为必经步骤。

## 下载次数与过期

分享还支持：

- 过期时间控制
- 最大下载次数限制

达到限制后，公开页会变成不可用状态，后端也会拒绝继续下载。

## 前端当前表现

当前前端公开页已经接入这些行为：

- 文件分享：下载、预览、密码门禁
- 文件夹分享：根目录浏览、文件下载、文件预览、grid/list 视图切换
- 公开链路跳过主站登录检查与 refresh 逻辑，避免误跳转到 `/login`

## 适合怎么理解这套能力

如果你只是要：

- 公开发一个文件
- 让别人下一个文件夹里的若干文件
- 给分享加密码或有效期

那现在这套已经够用了。

如果你要的是“完整公开文件浏览器”，包括：

- 子文件夹导航
- 公开 breadcrumb
- 整个目录树递归浏览

那当前实现还没到这一步。
