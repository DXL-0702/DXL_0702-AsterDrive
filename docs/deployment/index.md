# 部署概览

AsterDrive 只需要启动一个服务。
浏览器页面、公开分享页和 WebDAV 都在这个进程里，所以部署时最重要的是把这个服务长期稳定跑起来，并把数据保存好。

## 推荐方式

| 方式 | 适合谁 |
| --- | --- |
| [Docker](/deployment/docker) | NAS、单机、小团队、已有容器环境 |
| [systemd](/deployment/systemd) | 云主机、物理机、长期稳定运行 |
| 直接运行二进制 | 本地测试、临时验证 |

## 部署前先确认四件事

### 数据目录

重启或升级后，下面这些内容都要保留下来：

- 配置文件
- 数据库
- 本地上传目录

### 访问方式

正式上线时，建议通过 HTTPS 提供服务，并保持：

```toml
[auth]
cookie_secure = true
```

如果你只是本地或内网 HTTP 测试，可以暂时改成 `false`，等正式切到 HTTPS 再改回。

### WebDAV

如果你需要 Finder、Windows 或同步工具接入，部署时就要一起考虑：

- WebDAV 路径
- 反向代理
- 上传大小限制

### 存储位置

- 本地磁盘：部署最简单
- S3 / MinIO：适合对象存储场景

## 首次启动会自动完成这些动作

只要服务成功启动，就会自动完成这些准备：

- 生成默认 `config.toml`
- 连接数据库并自动更新数据库结构
- 自动创建默认本地存储策略 `Local Default`
- 初始化系统设置
- 启动回收站、锁、审计日志等清理任务

## 配置分三层

- `config.toml`：决定服务怎么启动
- 管理后台里的系统设置：决定 WebDAV、回收站、历史版本等运行行为
- 存储策略：决定文件落到哪里、怎么上传

## 上线后先验收这几项

1. 首页能正常打开并登录
2. `/health` 和 `/health/ready` 返回正常
3. 能创建文件夹并上传一个文件
4. 回收站恢复正常
5. 分享链接可以打开
6. 如果启用了 WebDAV，桌面客户端能成功连接
