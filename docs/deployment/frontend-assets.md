# 升级与浏览器缓存

AsterDrive 的页面、公开分享页和服务端由同一个程序提供。  
正常部署时，只要升级同一个镜像或同一个二进制，页面和服务端就会一起更新。

## 推荐做法

- Docker 部署：直接升级到新的镜像版本
- systemd 或单二进制部署：直接替换成新的 `aster_drive`
- 升级后刷新浏览器页面，再测试登录、上传、分享和 WebDAV

## 什么情况会出现版本不一致

如果你自己替换过页面文件，或者自己构建过页面资源，再单独升级服务端，就可能出现“页面还是旧的，服务已经是新的”这种半更新状态。

常见现象：

- 页面能打开，但按钮报错
- 新版本字段在页面里看不到
- 某些弹窗能打开，但提交失败

## 如果你自己替换过页面文件

如果你不是直接用现成镜像或二进制，而是自己构建或覆盖过页面文件，升级时要保证页面文件和服务端来自同一版本。

推荐顺序：

1. 备份 `config.toml`、数据库和上传目录
2. 停止旧服务
3. 一次性替换新的服务端和新的页面文件
4. 启动服务
5. 刷新浏览器缓存并做一次完整验收

## 反向代理不用单独拆静态站点

浏览器页面、公开分享页和静态资源都由同一个 AsterDrive 服务返回。  
反向代理通常只需要把整个站点代理到 AsterDrive，不需要再单独部署一套静态资源服务。

## 升级后页面看起来不对时先做这几步

1. 先确认服务已经升级到你预期的版本
2. 强制刷新浏览器页面
3. 如果你走的是 Docker，再确认容器是不是用了最新镜像
4. 如果你自己替换过页面资源，确认页面和服务端来自同一个版本
